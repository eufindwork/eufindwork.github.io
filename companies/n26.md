# N26 实习情报（Berlin）

> 范围：Software Engineering Intern（含 Backend / iOS / Android）、Werkstudent、Fintech Compliance / Risk 实习。N26 是德国最大原生 fintech / neobank 之一，持德国全行牌照（受 BaFin + ECB 直管）。

---

## 1. 公司与招聘概况

| 维度 | 信息 |
| --- | --- |
| 总部 | Berlin, Germany |
| 业务定位 | 全 EU 移动银行：账户 / 卡 / 支付 / 投资 / 加密 / 信贷；2025 起在荷兰试点抵押贷 / 借贷扩张被 BaFin 临时叫停 |

## New Grad / Junior 岗位

> 信息丰富度: ⭐⭐⭐⭐ (Berlin L2-L5 Levels.fyi 数据齐全 + 2026-05 实时更新, Medium / OphyAI 2026 面经丰富; BaFin 监管 + CEO 换帅信号明确, 但工程 hc 实际开放 (~20 active Berlin reqs))
> 区别于实习: 全职 entry-level (0-2 年经验); N26 内部 leveling 是 **L2-L5+** (Junior → Software Engineer → Senior → Lead) [来源 (实际打开过): https://www.levels.fyi/companies/n26/salaries/software-engineer/locations/berlin-metropolitan-region]; 应届 L2 (Junior) / L3 (Software Engineer)

- **常见岗位名**: Junior Software Engineer (L2 - Backend / iOS / Android)、Software Engineer (L3)、Junior Data Engineer / Analyst、Junior Compliance / Risk Analyst; 不使用 "Graduate" 命名, JD 直接写 "0-2 years" 或 "recent graduate welcome"
- **欧洲地点**: 工程核心 **Berlin Mitte**; 少量 **Vienna (Engineering hub)、Barcelona、Madrid**; Junior SWE 主要 Berlin; 当前 Berlin 约 **20 active reqs** [来源 (搜索结果): Glassdoor N26 Berlin Jobs 2026]
- **是否有独立 Graduate Programme**: **No** — N26 没有命名 grad scheme; Werkstudent → Junior Engineer 是最常见 new grad pipeline, Master Thesis 也常见
- **CEO 换帅 + BaFin 状态 (重大更新)**:
  - **Mike Dargan** (前 UBS Group Chief Operations & Technology Officer) **2026-04 正式上任 CEO**, 待 BaFin 监管批准 [来源 (搜索结果): https://thefintechtimes.com/ubs-executive-mike-dargan-appointed-ceo-of-n26-to-lead-next-chapter-of-growth/]

## 2. 岗位画像

### 2.1 Software Engineering Intern（Backend）
- 技术栈：**Kotlin（主力）+ Java**、**Spring Boot**、**AWS（重度）**、**Kubernetes**、**Kafka**、**Postgres + DynamoDB**、Terraform、Datadog。
- 架构：纯微服务 + event-driven（Kafka），合规链路独立部署（KYC / AML / fraud）。
- 团队方向：Payments core、Cards、Lending、Invest（股票 + 加密 + 基金）、Customer Identity / KYC、Risk & Fraud、Internal Platform。
- 期待：JVM 系列扎实、能理解事件驱动 + 幂等 / 一致性；对 banking 域 / 监管合规有兴趣。

### 2.2 iOS / Android Intern
- 技术栈：iOS = Swift（无 Obj-C），Android = Kotlin（无 Java legacy）；纯原生（不是 React Native / Flutter）；模块化 monorepo + 大量 Multi-module Gradle / SPM。
- 团队方向：Account、Card UX、Onboarding / KYC、Invest（含 crypto / stocks UI）、Internal SDK / Design System。
- 期待：写过完整 iOS / Android app，掌握现代架构（MVVM / TCA / Compose / SwiftUI）+ Concurrency。

### 2.3 Compliance / Risk / Fintech Operations Intern
- 团队方向：Financial Crime、AML、KYC、Fraud Ops、Regulatory Reporting；技术栈较轻（SQL + Excel + 部分 Python），但对 banking domain 知识要求高。
- 价值：在 BaFin 持续监管压力下，这块业务 HC 是 N26 内部少数仍在扩张的方向。

### 2.4 Werkstudent（在校长期 part-time）
- 是 N26 最常见的实习形式；20h/周；岗位覆盖几乎所有团队：Engineering、Data、Product、Marketing、HR、Legal、Compliance。

## 3. 面试流程

### 3.1 通用流程（实习 / 早期）

1. **Recruiter Call**（30 min）— casual 视频面，背景、动机（为什么选 fintech / N26）、价值观（**Courage / Simplicity / Empowerment**）、签证、薪资期望。
2. **Technical Screen**（45–60 min）
   - SWE：HackerRank challenge（6 quiz-style + 1 coding task，60 min）；或现场 live coding。
   - Mobile：take-home app（well-scoped 题目，测架构 / API / error states / best practices），通常 3–7 天提交。
3. **Take-home Review**（45–60 min）— 与 senior engineer 一起 review 提交的 take-home，深挖架构与设计决策。
4. **Live Coding / Pair Programming**（45–60 min）— banking 场景题：实现 transaction processor / balance calculator with concurrent ops / account management service。
5. **System Design / Architecture**（45–60 min，部分流程）— payment 幂等性、SAGA / outbox、Kafka 消息一致性、KYC 流程设计。
6. **Hiring Manager + Values**（45–60 min）— 是否对齐 N26 三大价值观；team fit。
7. **Final / Bar Raiser**（部分流程）

整体周期：**25–35 天**（Glassdoor 数据，比 Klarna 短）。

## 5. 签证与跨境工作

- **EU/EEA 学生 / 居民**：自由工作；首选群体。
- **非欧盟在德国在读**：student visa 自带打工权（强制实习不计入 120/240 上限）。
- **海外申请者**：实习不主动 sponsor，N26 招聘官在 recruiter call 中会明确确认 EU 工作权。
- **全职毕业生**：会 sponsor EU Blue Card（2025 阈值约 €45,300/年，N26 SWE 起薪通常达标）；fintech 行业整体 sponsor 友好。
- **远程**：N26 内部允许 Germany / Vienna / Barcelona 之间跨办公室，但跨欧盟外远程不允许。

## 6. 申请策略 & 时间线

| 阶段 | 时间 | 备注 |
| --- | --- | --- |
| Job posting | 全年滚动；2025–2026 hc 紧缩，部分团队 hiring freeze | 关注 n26.com/careers + LinkedIn |
| Application screening | 1 周 | Resume 关键词：Kotlin / Spring Boot / Kafka / AWS / Swift / Kotlin (Android) / Banking domain |
| Recruiter call | 1 周 | 确认 EU 工作权 + 三大 values 对齐 |
| Take-home / HackerRank | 1–2 周 | Take-home 质量是评分核心 |
| Tech rounds | 2–3 周 | Live coding + system design 各一轮 |
| HM | 1 周 | 行为 + values |
| Offer | 25–35 天 | 比 Klarna 短，比 Zalando 略快 |

申请技巧：
- **Take-home 是评分核心**：写出 production-quality 代码（单测、错误处理、清晰架构、README 解释 trade-off）能直接拉开差距。
- 准备 banking domain 问答：账户 / 卡 / 支付清算 / KYC / AML / PSD2 / Open Banking 基础概念。
- N26 三大价值观要在故事里能映射到具体行为：**Courage**（敢挑战 / 推进风险但合规决策）、**Simplicity**（产品 / 代码 / 流程的简化案例）、**Empowerment**（团队赋能 / 知识共享）。
- 对**监管 / 合规风险**有意识：能讨论 BaFin / ECB / PSD2 / DORA / MiCA / GDPR 等会显著加分（即使是工程岗）。
- 2025–2026 因为 BaFin 监管 + 换帅，正在收紧 hc；非工程岗位（Compliance / Risk）反而是少数仍在扩招的方向。

## 8. 关键风险 / 提示

1. **BaFin 监管不确定性**：2025 重启 special monitor + 提高资本要求 + 荷兰抵押停售；team budget 受冲击；实习 hc 紧缩。
2. **CEO 换帅过渡期**：Co-founder Tayenthal 退场，UBS 高管 Mike Dargan 2026-04 上任，战略与组织架构很可能调整；offer 后入职日期 / 团队归属易变动。
3. **不 sponsor 实习签证**：海外候选人靠 Master Thesis 或先在德国念书；招聘 recruiter 会明确确认。
4. **Take-home 高门槛**：take-home 评分极严，需要单测 + CI + README + 设计 trade-off 说明；不要把它当成"作业级别"。
5. **Banking domain 加分明显**：在简历或面试故事里能聊清楚 KYC / AML / payment rail / PSD2 会显著正向。
6. **三大 Value 要内化**：Courage / Simplicity / Empowerment 是每个 round 都被打分的项；不像 Spotify Values 那么张扬，但权重一样大。
7. **不同岗位 hc 不均**：工程 / 产品 / 数据团队 2024–2025 普遍紧缩；Compliance / Risk / Financial Crime 仍在扩张。
