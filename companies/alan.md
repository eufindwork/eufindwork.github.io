# Alan (巴黎) — 实习情报

> 法国/欧洲 healthtech 标杆, 既做健康保险又做应用平台 (B2B2C). 文化独特 ("no meeting", 完全透明薪资, written-down decisions), 工程招聘严苛但流程透明且 candidate-friendly. M2 6 个月毕业实习有明确开口.

## 2. 实习岗位

Alan 的实习岗位**面向 M2 毕业 stage**, 单一明确 funnel:

| 岗位名 | 团队 | 时长 | 关键词 |
|---|---|---|---|
| Software Engineer Internship (6 months) | Engineering (Product 端, Tech Foundations 端均有) | 6 个月 | Python/Flask + React/Redux/React Native, Postgres, 端到端 ship |

部分年份/特殊招聘也有:
- Product Engineer 实习
- Data / ML Engineer 实习 (较稀少, 主要做 health risk modeling, content moderation, member support 自动化)
- Security Engineer 实习

**地点**: Paris (主要), 远程 (FR/BE/ES 内可商议).

## 4. 招聘流程

Alan 整体招聘流程对 senior 是 ~33 天, 实习相对压缩. 主结构来自 Medium 文章和官方 hiring page:

| 轮次 | 内容 | 时长 | 形式 |
|---|---|---|---|
| 1. Screening | 双向了解, 一半时间 candidate 提问 | 30 min | Video |
| 2. Technical Interview (CoderPad) | "**不是 algorithm puzzle, 不是脑筋急转弯**" — 实现一个接近日常工作的小程序; 共享编辑器 (coderpad.io) | 60 min | Online (coderpad.io) |
| 3. Alaner Fit Interview | 创始人/资深 Alaner 主持; 考察 mission-driven mindset, growth mindset, 沟通, "take a step back" 能力 | 45 min | Video |
| 4. Checkpoint | 候选人反馈与提问机会 | 短 | Async / Call |
| 5. Reference Calls | 候选人提供的 reference (Alan 不背对背调查) | 可选 | 电话 |
| 6. Alan Day | 半天到一天 onsite (Paris) 或 video 全套 | 4-5 小时 | Mix |

**Alan Day 由 4 个 sub-interview 组成**:

| Alan Day 子轮 | 内容 | 时长 |
|---|---|---|
| **Past Project** | 选一个你深度参与过的项目, 讲设计/挑战/决策/技术债; 考察沟通 + 技术深度 | 60 min |
| **Feature Design** | 给一个 Alan 实际可能要做的新 feature, 讨论怎么设计/拆分/上线 — 像真实 brainstorm | 60 min |
| **Ask Me Anything** | 角色反转, candidate 问公司 | 30 min |
| **Debrief** | 收集候选人对流程的反馈 | 15+ min |

**Decision**: Alan Day 后 **最多 48 个工作小时内出 offer/拒信**.

**特别说明** (重要):
- **No take-home assignment** — Alan 官方明说不出 6-7 小时 take-home, 因为不公平/不 inclusive
- 流程**结构化**, 每轮后都给 candidate 详细反馈 (不论过与不过)
- 用面试评分网格 + 多元面试官, 借鉴 FB/Google 的 unconscious bias training
- 没有 algorithm puzzle, 但 feature design 的"系统思维"考察很深

## 7. 文化, 工作环境, 软情报

Alan 的工程文化是 Paris 区最特别的之一, 关键特征:

- **No meeting** — async + written-down 文档驱动; 实际上还是有些 sync, 但 default 是 async
- **Salary fully transparent** — 全员都能查彼此薪水; 没有 negotiation, 按 level 自动定
- **No job titles** — 不分 "Senior SDE I/II/III", 只有几个等级
- **Pair programming + code review** 文化重, mentorship rigorous
- **Role buddy** 制 — 每个新人 (含实习) 配一个 dedicated engineer 带前 weeks/months
- **Rigorous hiring** — 进来难, 进来后被很认真对待
- **Engineering principles**: written-down (Medium 上 Charles Gorintin 写的); 强调 ownership, trust by default, async by default
- **Glassdoor 42% positive interview** — 不算高, 但口碑是 "challenging but human, transparent feedback"
- **远程友好** — 但 intern 一般倾向 Paris onsite 以便 mentorship

## 8. 申请策略与时间线建议

| 步骤 | 建议 |
|---|---|
| 1. 投递时间窗 | H1 2026 起的 stage **最晚 10-11 月投** (Alan 实习滚动招但量小, 越早越好) |
| 2. 必备先决条件 | **真的把全栈 app 部署到生产** — 不是 hello-world, 是真用户 用过的. 这是面试官第一个会问的 |
| 3. CV 重点 | 1-2 个端到端项目 (Flask 或 FastAPI + React + Postgres, 部署到 Vercel/Fly.io/Render), 配 GitHub + Live URL |
| 4. CoderPad 准备 | **不是 LeetCode**. 练习真实场景: "给你 5 个文件的 mini Flask app, 找 bug 修复", "实现一个简单 CRUD endpoint" — 用 Python + 现成 framework, 不裸写 |
| 5. Feature Design 准备 | 假设题: "如何设计 Alan 的 claim submission flow?" — 用户故事 → UI 草图 → API 设计 → DB schema → edge case → 上线 plan. 看 Alan 公开博客上的工程文章 |
| 6. Past Project 准备 | 选**一个**项目讲深: 为什么做, 设计抉择, 你的具体贡献, 最大的技术债, 之后怎么改 |
| 7. Alaner Fit | "Mission-driven" — 想清楚你为什么对 health/insurance 有热情. 套话会被识别 |
| 8. 准备问题 | Screening / AMA 都会大量留时间给候选人; 准备 10+ 高质量问题 (关于文化, async, no-meeting 实际怎么运作, mentorship) |
| 9. 备选 | 同档位巴黎健康/保险 tech: Doctolib (更大), Lifen, Qare, Withings; 同档 fast-growing startup: ContentSquare, Datadog, Mirakl, Aircall, Spendesk |
